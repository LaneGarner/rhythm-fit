Activity Flow Polish

Design Rationale and Manual QA

A compact review guide for the activity library, execution cards, supersets, rest countdown behavior, completion feedback, and timer notification reliability.

Norman UX Frame

These changes reduce hidden state and make the available action map more directly to the user's workout task: find an activity, perform a set, understand what is complete, and control whether rest starts automatically.

Affordances

Rows scroll, toggles toggle, wheel fields invite picking.

Signifiers

Names, type labels, counts, and status text replace decorative activity glyphs.

Feedback

Completed sets use stronger color, borders, and explicit labels.

Mapping

Reps step by 1, weight by 5, matching common gym adjustments.

Constraints

Rest auto-start requires both global and per-activity enablement.

Error Prevention

Superset unlink asks for confirmation and preserves existing workout data.

Per-activity rest countdown Auto behavior is local and reversible, while manual timer controls stay available. Enable global auto rest timer in Settings. Superset rest countdown gate A superset should not auto-start if any included activity opted out. Disable rest countdown for one activity in a superset. Complete a superset round. No auto rest countdown starts until every superset activity has rest countdown enabled and global auto rest is enabled.
Superset visual cleanup Superset execution cards should feel like regular execution cards. Open a superset execution flow and compare card styling to regular activity execution. Cards use neutral card styling without blue rails; connectors are neutral.
Unlink superset Destructive group change has a clear confirmation and preserves data. Open a superset, tap Unlink Superset, cancel once, then confirm. Cancel does nothing. Confirm splits activities, preserves sets/order/completion, and returns to the prior screen.
Completion feedback Progress should be visible without interpreting color alone. Complete sets one by one in regular activity execution. The header shows x of y sets complete. Completed sets have success styling and status text. All sets complete shows an Activity Complete banner.

Timer Notification QA

Timer notifications now separate generic timer completion from rest completion IDs, cancel stale opposite IDs before scheduling, cancel both IDs when timer notifications are disabled or the timer stops, and schedule through the default Android channel with high priority.

Foreground

  1. Enable notifications and timer completion in settings.
  2. Start a short manual countdown while the app is open.
  3. Wait for completion.

Expected: in-app timer feedback appears; foreground banner may be suppressed by design.

Background

  1. Start a 20-30 second countdown.
  2. Send the app to background.
  3. Wait for completion.

Expected: local notification appears with sound if permissions and OS settings allow it.

iOS Lock Screen

  1. Use a physical iPhone, not Simulator.
  2. Confirm iOS Settings -> Rhythm -> Notifications allows Lock Screen, Banners, and Sounds.
  3. Disable Focus modes that silence the app.
  4. Start an auto-rest countdown, lock the phone, and wait.

Expected: "Rest complete" notification is delivered on the lock screen when iOS permits delivery.

Limit What to verify
Permissions Notifications, lock-screen display, and sounds must be granted at OS level.
Focus / Do Not Disturb Focus can delay or hide active notifications unless the app is allowed.
Silent mode and volume Local notification sound behavior depends on iOS sound settings; in-app audio uses the app audio mode.
Expo Go Use a development or production build for reliable notification validation.
Android exactness Android may apply timing variance without exact alarm privileges on some devices.

Regression Checklist

Data Integrity

  • Duplicate regular and superset sets; verify copied values.
  • Unlink a superset; verify activities keep original dates, sets, order, and completion.
  • Edit reps/weight; leave and reopen activity; verify persistence.

Navigation

  • Back/Edit controls still work from regular and superset execution.
  • Share preview still renders with check/status icons.
  • Activity library modal save/cancel still works after scrolling.

Accessibility

  • Switch exposes checked state for Auto Rest Countdown.
  • Set complete buttons announce complete/incomplete state.
  • Icon removal does not remove the text label needed to identify any activity.

Verification

Static verification command: npm run typecheck. Manual verification should prioritize small iPhone viewports, iOS physical-device notification delivery, and real workout flows with multiple set types.
